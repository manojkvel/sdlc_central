# SDLC Central — Team Productivity Hub

## What This Is
A central repository of 60 skills and 30 composable pipelines covering the full SDLC lifecycle. Agent-agnostic core with adapters for Claude Code, Cursor, Copilot, Windsurf, Cline, Aider, Gemini, Antigravity, and AGENTS.md.

## Repository Structure
- `skills/` — All 60 skill definitions (source of truth: `skill.yaml` + `prompt.md`)
- `adapters/` — Per-agent output generators (claude-code, cursor, copilot, windsurf, cline, aider, gemini, antigravity, agents-md)
- `pipelines/` — Composable workflow chains per role (agent-agnostic YAML)
- `registry/` — Skill catalog and role mappings
- `templates/` — Per-role CLAUDE.md templates (also in `adapters/claude-code/templates/`)
- `setup/` — Installation and update scripts (support `--agent` flag)
- `config/` — Default quality gate and scoring configurations
- `docs/` — Guides, references, per-agent quickstarts
- `package/` — npm distribution

## Key Commands
- `bash setup/install.sh` — Interactive installer (select agent + role)
- `bash setup/install-role.sh <role> --agent <agent>` — Install for a specific role and agent
- `bash setup/install-all.sh --agent <agent>` — Install all 60 skills
- `bash setup/update.sh` — Update to latest preserving config
- `/run-pipeline <role>/<pipeline>` — Execute a pipeline workflow

## Development Guidelines
- **Source of truth**: `skills/<name>/skill.yaml` (metadata) + `skills/<name>/prompt.md` (agent-agnostic prompt)
- **Legacy**: `skills/<name>/SKILL.md` kept for backward compatibility, generated by claude-code adapter
- Pipeline YAML files define step chains with gates and dependencies — already agent-agnostic
- Always update `registry/catalog.yaml` when adding/modifying skills
- Test with `bash setup/install-role.sh <role> --agent <agent>` in a temp directory
- Adapters live in `adapters/<agent>/adapter.sh` — each transforms universal format to agent-native files
